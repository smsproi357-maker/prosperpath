# Feature: Global UI & Design System (`styles.css`)

## 1. Purpose
Defines the overarching "Wealth-Tech Aesthetic" of ProsperPath. It establishes a unified, premium visual language (dark theme with gold accents) across all domains, ensuring visual continuity without relying on massive external CSS frameworks.

## 2. Core Functionality
- **CSS Variable Registry**: Centralizes all color hexes, typography stacks (Inter/Playfair), spacing multipliers (var(--space-X)), and border-radii into the `:root` pseudo-class for immediate global propagation.
- **Deep Vignette Background**: Injects a fixed `body::before` radial gradient creating a subtle, noise-textured infinite dark canvas beneath all elements.
- **Universal Layout Grid**: Establishes responsive `.container` bounds (`max-width: 1280px`) and `.grid-X` utility classes mapping directly to CSS Grid templates.
- **Component Primitives**: Styles foundational atomic elements including `.btn-primary` (gold buttons with drop shadows), native navigation bars (`.header`), and `.blog-card` wrappers.

## 3. Detailed Logic
- **Color Variables**: 
  - Primary Background: `#050506` (Near absolute black).
  - Primary Accent: `#F5C842` (Muted warm gold).
  - Surfaces: Leverages translucent `rgba(255,255,255, 0.05)` rather than solid grays to allow the background vignette to bleed through components slightly.
- **Header Scroll State**: Transitions `.header.scrolled` by shrinking `padding` from `var(--space-4)` to `var(--space-3)` and intensifying the `backdrop-filter: blur(32px)` upon vertical Y-axis scrolling.
- **CSS Animations & Transitions**: 
  - Sets standard `--transition-base: 0.3s ease;`.
  - Maps complex hover states (e.g. `.blog-card:hover`) to shift `transform: translateY(-8px)` paired tightly with a glowing box-shadow array to create physical depth.

## 4. User Flow
1. User loads any HTML page in the ecosystem.
2. The browser parses `styles.css` synchronously.
3. The root variables instantiate the dark-mode aesthetic instantly.
4. User clicks a navigation `.dropdown-panel`. The CSS handles the `opacity 0 -> 1` and `transform translateY(-6px) -> 0` natively without JavaScript overhead.
5. User resizes the window. CSS clamp functions (`clamp(2.5rem, 6vw, 4.5rem)`) dynamically step down typography sizes fluidly.

## 5. Inputs / Outputs
- **Inputs**: Raw DOM Nodes.
- **Outputs**: Rendered pixel coordinates, computed color maps, and structural flow boundaries painted to the canvas.

## 6. Edge Cases
- **Z-Index Warfare**: Strictly defines `--z-dropdown: 100`, `--z-sticky: 200`, `--z-modal: 300`, and `--z-tooltip: 400` globally to prevent overlapping component conflicts standard in SPA development.

## 7. Dependencies
- **Google Fonts**: Depends on the external fetching of `Inter` (Sans-serif) and `Outfit`/`Playfair Display` (Serif/Display) to render properly.

## 8. UI / UX Behavior
- **Text Gradients**: Implements webkit-specific `-webkit-background-clip: text;` atop linear gradients to render the gold-shimmer text prominent in hero sections.
- **Minimalism**: Removes native outlines (`outline: none`) but explicitly replaces them within `:focus-visible` states mapping to `--color-accent` to preserve strict accessibility standards.

## 9. Future Improvements
- Refactor massive single-file `styles.css` (6000+ lines) into discrete modular files (e.g. `layout.css`, `typography.css`, `components.css`) specifically imported to reduce scroll fatigue during developer maintenance.

## 10. System Role
The visual nervous system of the application. Without it, the application functionally works, but lacks the core psychological trust layer that the "Premium Institutional" aesthetic generates for the user.
