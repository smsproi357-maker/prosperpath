# Feature: Prosporous AI Assistant Widget (`ai-widget.js`)

## 1. Purpose
Provides an omnipresent, context-aware LLM chatbot (Prosporous) docked to the bottom right of the screen. It allows users to query financial concepts, summarize active articles, or ask portfolio questions without breaking their current workflow or leaving the page.

## 2. Core Functionality
- **Contextual Scraping**: Automatically identifies the current page (e.g. Portfolio vs. Crypto Detail) and invisibly extracts specific DOM data (Asset prices, Total Holdings) to feed the LLM as hidden context.
- **Serverless LLM Proxy**: Routes chat requests securely through a Cloudflare Worker (`WORKER_API_URL`), bypassing client-side API key exposure while leveraging OpenRouter models (like `zhipu/glm-4.5-air`).
- **Tavily Search Hook**: Optionally integrates Tavily API to execute live web searches appending external results directly to the LLM's system prompt.
- **Session Management**: Handles local storage creation, rendering, and cloud synchronization (`syncSessionsToCloud`) of chat history arrays.

## 3. Detailed Logic
- **Context Constructor (`getPageContext`)**:
  - Drops into specific logical branches via `detectPageType()`.
  - If `crypto-detail`: Grabs `.detail-title h1` and `.price-main`.
  - If `portfolio`: Traverses global `window.portfolioData`, maps complex Holding objects into a clean array structure (`name`, `ticker`, `value`), and calculates `totalHoldingsValue`.
  - Injects this massive JSON directly into the final system prompt sent to the LLM.
- **Action Pills (`initGuideAssistant`, etc.)**:
  - Fires `setTimeout` delays upon page load.
  - Generates HTML buttons (`data-action="summarize"`) directly inside the chat window.
  - Binds click listeners that synthesize pre-written prompts and automatically trigger `sendMessage()`.
- **Message Rendering (`appendMessage`)**:
  - Employs basic Regex handlers to convert raw `**text**` to HTML `<strong>` strings securely parsing the LLM markdown outputs into readable HTML blocks.

## 4. User Flow
1. User explores the Portfolio page and notices the floating gold AI icon.
2. User clicks the widget (`toggleChat`), sliding up the Chat Window.
3. User types: "Am I too exposed to Tech?"
4. `sendMessage()` intercepts, bundling the user text with the `getPageContext()` payload containing the full stock portfolio matrix.
5. AI processes the combined payload via the Cloudflare Worker.
6. A typewriter-style effect renders the streamed (or chunked) AI response advising the user on their specific mapped stock allocations.

## 5. Inputs / Outputs
- **Inputs**: User `<textarea>` strings, localstorage keys (`prosporous_sessions`, auth tokens), DOM innerText scrapers.
- **Outputs**: Asynchronous `fetch` calls to `neurowealth-worker.smsproi357.workers.dev`, DOM injections into `.prosporous-messages`.

## 6. Edge Cases
- **Legacy Migrations**: Explicitly checks for old history keys (`prosporous_chat_history`), bundles them into a new `legacy-Date.now()` session id, and nukes the old key preventing local storage bloat on version upgrades.
- **Cloud Sync Collisions**: Syncs arrays `user/chat` endpoints, utilizing simple ID Set matching (`localIds.has(s.id)`) to merge downstream cloud history into the local client without duplicating identical chains.

## 7. Dependencies
- **CloudBackend (Workers)**: Dependent on `WORKER_API_URL` availability to securely manage the upstream OpenRouter LLM fetches without exposing primary API keys.
- **Global Auth (`getAuthToken`)**: Relies on the user being logged in to execute the cloud history sync routines.

## 8. UI / UX Behavior
- **Floating Mechanics**: Anchored `position: fixed; bottom: 20px; right: 20px;` persistently maintaining z-index dominance across all views.
- **Fullscreen Toggle**: Allows the user to expand the chat module from a small corner widget into a centralized massive application overlay if deep research is required.

## 9. Future Improvements
- Transition from manual regex mappings to a robust library like `marked.js` to perfectly render complex LLM markdown tables, blockquotes, and code blocks within the chat UI.
- Implement true WebSockets for the LLM stream rather than standard HTTP polling to allow for smoother text rendering.

## 10. System Role
The primary intelligence interface. It converts a static dashboard into a dynamic "Pair-Investing" teammate capable of explaining what the user is looking at contextually without manual copying and pasting.
