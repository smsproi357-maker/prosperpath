# Feature: Progress & Certification Tracking

## 1. Purpose
Tracks, saves, and visually reflects the user’s continuous educational journey across domains. It manages global module unlocks, localized completion statistics, and calculates final macro-certifications (e.g. Structural Competency).

## 2. Core Functionality
- **State Persistence**: Serializes and stores completion states and mastery scores locally to maintain sessions across multiple browser refreshes.
- **Progress Visualizers**: Computes live percentage trackers on both the micro (individual module screen) and macro (domain hub dashboard) scale.
- **Certification Gate**: Automatically polls verified module counts to unlock comprehensive "Domain Certifications" providing conceptual diplomas.
- **Routing Engine**: Powers the 'Set Your Learning Path' algorithms allocating specific sequence orders based on onboarding configuration states.

## 3. Detailed Logic
- **Micro Progress (`computeProgress()`)**:
  - Dynamically calculates internal module completion: `pct = Math.round((visited_screens / total) * 85)`.
  - Conditionally maps the final 15% exclusively to passing the Mastery Assessment (`STATE.assessment.passed` jumps `pct` to 100).
- **Macro Persistence (`localStorage`)**:
  - Interfaces heavily with `localStorage.getItem('pp_mm_modules')`.
  - Object structure tracks explicit key mapping `{ "1.1": { verified: true, mastery: 80, status: "Verified" } }`.
- **Dashboard Assembly (`market-mechanics.html`)**:
  - Fetches the `MODULES_DB` object representing the canonical list of modules.
  - Iterates and deeply merges the immutable database listing with the mutable `pp_mm_modules` local progress object.
  - Generates HTML badges reflecting `Not Started`, `In Progress`, or `Verified` mapping natively to specific CSS variables depending on state.
- **Certification Computation (`updateCertificationStatus`)**:
  - Traverses the synced module mapping counting keys where `module.verified === true`.
  - Updates the DOM string (e.g., `3 / 5 Verified`).
  - Triggers a visual state change locking or unlocking the "Claim Certification" overlay interface when threshold equals total length.

## 4. User Flow
1. User interacts within an individual module, watching the linear top progress bar fill up synchronously as they scroll screens.
2. User passes the assessment. The internal variable writes heavily to `localStorage`.
3. User returns to the overarching `market-mechanics.html` dashboard.
4. Dashboard intercepts the pageload, scans `localStorage`, and updates the UI specifically changing the card border to gold and activating a verified checkmark.
5. User completes all 5 domain modules.
6. User clicks 'View Certificate', opening a cryptographic-style summary modal attributing their success.

## 5. Inputs / Outputs
- **Inputs**: Abstract `localStorage` objects, `visitedSections.size` integers during micro runtimes.
- **Outputs**: Scaled CSS `width: X%` commands applied to DOM progress lines, dynamically interpolated status strings injected into headers, conditionally revealed DOM arrays (like the Recommendation list).

## 6. Edge Cases
- **Broken JSON Catchers**: Enforces strict `try/catch` wrappers around all `JSON.parse(localStorage)` calls gracefully substituting empty `{}` default objects upon finding corrupted strings to prevent total application crash.
- **Legacy Migrations**: Explicitly handles null values on older module structure attempts mapping undefined mastery scores universally back to `0%`.

## 7. Dependencies
- **Core Educational HTML Sheets**: Dependent implicitly on individual module files securely calling internal mapping updates upon quiz submission completion.

## 8. UI / UX Behavior
- **Top Bar Tracker**: Displays an explicitly smooth `width: .4s ease;` animation sliding a gold line completely across the application ceiling visualizing internal depth intuitively.
- **Domain Hub Cards**: Progressed modules adapt specifically thicker, colored borders (`mm-module-card.verified`).
- **Certification Modal**: Mimics institutional diploma aesthetics deploying muted backgrounds, UUID visualization string generations, and dark 'gold' accents enforcing premium completion rewards.

## 9. Future Improvements
- Shift from client-local tracking inherently insecure to dedicated secure database representations via Supabase bindings avoiding potential user modifications to HTML standard memory files.
- Enable macro-progress cross comparisons displaying "Community Completion" distributions on specific quizzes natively onto the global UI dashboards.

## 10. System Role
Functions as the macro-economic backbone of the broader gamified educational platform; ensuring user retention by implementing measurable progression layers seamlessly operating in the background.
