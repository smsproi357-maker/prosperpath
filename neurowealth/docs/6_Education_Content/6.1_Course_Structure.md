# Feature: Educational Course Structure

## 1. Purpose
Provides a structured, interactive curriculum (ProsperPath Insights) teaching users institutional market mechanics. It breaks complex financial concepts into consumable "Domains" and specific "Modules," focusing on interactive retention rather than passive reading.

## 2. Core Functionality
- **Domain Organization**: Groups learning into overarching domains (e.g., Domain I: Market Structure, Domain II: Market Context).
- **Module State Machine**: Each individual module (e.g., `module-1-1.html`) is internally structured as a self-contained Single Page Application (SPA) driven by a JavaScript state machine.
- **Progressive Disclosure**: Forces sequential reading logic initially; users cannot jump ahead to sections they haven't unlocked, ensuring linear consumption of foundational concepts safely.
- **Clarify Panel**: Provides contextual overlaid dictionary definitions for complex terms directly within the flow without losing place.

## 3. Detailed Logic
- **Module Architecture**: Embeds a constant `STATE` object tracking `currentSectionId` and `visitedSections` (e.g., Entry -> Concept -> Simulation -> Test -> Reality -> Reflection -> Assessment).
- **Navigation (`goToSection`)**: 
  - Bounds checks requested section IDs against `STATE.visitedSections.has(sid)` or `sid <= max_visited + 1`.
  - Maintains `navHistory` arrays to support localized contextual browser-like "back" buttons without actually triggering a page reload.
- **Screen Renderers**: Employs pure functional string interpolation (`buildS1()`, `buildS2()`, etc.) directly re-writing `innerHTML` of the `#screen-container` upon state changes.
- **Contextual Dictionary (`openClarify(topicId)`)**: Injects hardcoded semantic objects (e.g., `CLARIFY_DATA.bid`) into an absolute positioned, animated DOM slider out from the right edge.

## 4. User Flow
1. User enters via Domain Hub (e.g. `market-mechanics.html`) and selects a specific Module.
2. User is greeted by the Entry Screen outlining time commitment and topics.
3. User navigates sequentially through Conceptual screens utilizing "Continue" CTAs.
4. User completes interactive mid-module Decision Tests (checking comprehension).
5. User enters subjective Reflection text fields.
6. User completes the final Mastery Assessment.
7. Upon pass, user is redirected back to the Domain Hub with verified status unlocked.

## 5. Inputs / Outputs
- **Inputs**: User click events, Navigation toggle commands, Reflection text box inputs.
- **Outputs**: Dynamic HTML re-rendering of internal module segments, visually updated native progress bars, DOM overlay state toggles.

## 6. Edge Cases
- **Deep Linking**: Modules do not support deep URL linking to specific internal sections; refreshing resets the user to the Entry block to enforce sequence review.
- **Mobile Readability**: Detects narrow viewports dynamically shifting the left-hand section navigator into a hidden hamburger-menu slide-over drawer to maximize content reading space.

## 7. Dependencies
- **Core Styles**: Extensively utilizes native layout conventions derived entirely from the global `styles.css`.
- **LocalStorage API**: Downstream coupling to write module completion objects (`pp_mm_modules`).

## 8. UI / UX Behavior
- **Non-Scrolling Stage**: The primary interface is explicitly constrained to `100vh`. Only the central `#main-stage` text area features `overflow-y: auto`, providing a focused, app-like reading experience.
- **Dark Mode Aesthetics**: Built atop the `var(--m-bg)` dark theme utilizing gold (`var(--m-gold)`) to highlight active elements and progression cues.
- **Animation Framework**: Screens utilize rapid `.2s` CSS Keyframe `fadeIn` translations to visually transition conceptual stages smoothly.

## 9. Future Improvements
- Externalize the massive localized HTML interpolated strings into discrete JSON payload fetches or Markdown files to heavily reduce individual file sizes.
- Implement server-side synchronizations for module state allowing cross-device pause and resume capabilities.

## 10. System Role
Acts as the human-layer foundational bridge. By educating the user accurately on order-flow mechanics, it builds user trust and understanding required to responsibly wield the more complex automated Backtest and Portfolio engines.
