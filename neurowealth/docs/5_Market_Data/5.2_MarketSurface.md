# Feature: Market Surface & Asset Detail

## 1. Purpose
Provides institutional-grade interactive charting and AI-curated asset analysis in a dedicated fullscreen view. It translates abstract OHLCV (Open, High, Low, Close, Volume) data arrays into both a visual heatmap and an actionable textual summary format.

## 2. Core Functionality
- **High-Performance Charting**: Wraps the `LightweightCharts` WebGL engine to render thousands of candlestick and volume nodes cleanly without browser latency.
- **Automated Technical Context**: Dynamically evaluates the raw chart arrays to output plain-english tags dictating Trend, Range expansion, Volatility, and Key Resistance levels.
- **Infinite Left-Scroll Backfill**: Implements advanced subscription events to seamlessly fetch and prepend historical data exactly when the user pans near the oldest visible node limit.
- **AI Synthesis Feed**: Maps external sentiment scores, dynamic verdict analysis, and related news feeds natively to the specific queried asset wrapper.

## 3. Detailed Logic
- **Context Derivation Engine (`deriveContext`)**:
  - Receives the raw mathematical OHLCV array.
  - **Trend**: Calculates a 20-period Simple Moving Average (`calcSMA`). Evaluates if the last close sits >0.5% above/below this line to label standard `Bullish`/`Bearish` tags.
  - **Range**: Scans the bandwidth (`High - Low`) of the last 20 candles vs the previous 20 to detect Bollinger-style expansion/contraction multipliers (`expansion > 1.3`).
  - **Volatility**: Computes true Average True Range (`calcATR`). Benchmarks ATR as a percentage representation of current price.
- **Renderer (`renderChart`)**:
  - Initializes the canvas explicitly overriding native `LightweightCharts` styling tokens with custom ProsperPath hex constants (`TOKENS.bg`, `TOKENS.upVolume`).
  - Binds independent `addCandlestickSeries` and `addHistogramSeries` to identical time scales.
  - Extends a universal `ResizeObserver` listener on the canvas root to forcefully scale the internal engine vectors dynamically when side-panels collapse.
- **Backfill Engine (`subscribeVisibleLogicalRangeChange`)**:
  - Listens to every user drag event across the X-axis.
  - Drops a trigger mathematically set at `BACKFILL_TRIGGER_FRACTION = 0.15` (15% distance from the oldest loaded candle).
  - Pauses UI, queries `fetchYahooOHLCBefore(timestamp)`, scrubs duplicates, and merges Arrays `[...newCandles, ...allData]`. Native API `setData()` handles the clean graphical prepends.

## 4. User Flow
1. User navigates to `market-detail.html?symbol=AAPL`.
2. The page loads displaying a large structural skeleton while executing async payloads.
3. Once initialized, the user sees a combined OHLC chart set to a localized `1D` timeframe.
4. User pans backwards (to the left) on the chart. At the edge, the system silently fetches years of older historical data, extending the line forever backwards.
5. User clicks '15m'. The chart destroys the active cache block, queries Yahoo Finance intraday nodes, sets the time axes to visible, and redraws the graph perfectly.
6. The user immediately reviews the "Context Strip" below the chart to see if volatility is marked "Elevated" translating the chart into readable text.

## 5. Inputs / Outputs
- **Inputs**: Core URL query strings defining symbols, `fetchYahooOHLC` JSON data objects containing structured time-series elements.
- **Outputs**: Rendered HTML string chunks `buildCard(opts)`, instantiated WebGL canvas layers via TradingView's open source `LightweightCharts`.

## 6. Edge Cases
- **Library Unavailability**: Prevents severe crashing if the external unpkg CDN drops by deploying safe HTML fallbacks indicating "Chart library not available" internally if `window.LightweightCharts` resolves falsely.
- **Data Exhaustion Capping**: The backfill loop accurately intercepts duplicated data outputs from the API mapping, triggering an explicit boolean `noMoreLeft = true` switch preventing infinite null loops when assets have zero further history.
- **Non-Standard Assets**: Allows overriding the volume rendering engine (`showVolumePanel = false`) explicitly for variables like Forex pairings or distinct Indices which don't carry native volume metrics naturally cleanly mapping the canvas arrays cleanly regardless.

## 7. Dependencies
- **TradingView's Lightweight Charts**: `lightweight-charts.standalone.production.js` bound entirely as the foundational visual engine.
- **script.js**: Dependent upon API fetch routing instructions (`fetchYahooOHLC`, `fetchYahooOHLCBefore`) configured globally for proxy bouncing.

## 8. UI / UX Behavior
- **Empty States**: Specifically avoids throwing raw errors; invokes `showChartEmpty` injecting visually native SVG style telemetry signals alongside localized retry buttons cleanly if the market happens to be shut or heavily lagging.
- **Crosshair Mapping**: Injects dashed CSS stroke overlays matching local `TOKENS.crosshair` coordinates smoothly scaling the background Y-axis nodes with active hover intersections cleanly visually grouping to standard industry behaviors.

## 9. Future Improvements
- Inject natively calculable standard overlays (like EMAs or Bollinger Bands) visibly directly onto the primary candle series rather than existing merely as headless mathematical variables generating text strings.
- Add advanced drawing tools mapped to an overarching localstorage payload array holding local annotations natively connected over discrete user sessions.

## 10. System Role
Functions as the primary analytical "deep dive" module for all independent asset variables in the ecosystem. It provides the terminal-like graphical horsepower bridging pure fundamental data points into visual structural analysis.
