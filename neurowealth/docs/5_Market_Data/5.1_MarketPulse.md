# Feature: Market Pulse / Live Feed

## 1. Purpose
Provides a macroscopic, unified ticker of real-time market data across multiple asset classes (Crypto, Stocks, Commodities). It serves as an ambient, non-intrusive dashboard component that quickly conveys broad market direction without requiring users to actively search for specific tickers.

## 2. Core Functionality
- **Cross-Domain Fetching**: Aggregates top data natively from APIs like CoinGecko (for Crypto) and Yahoo Finance (for traditional equities and commodities).
- **Categorical Switching**: Splits data structurally into distinct, togglable arrays (`Crypto`, `Stocks`, `Commodities`).
- **Resilient Caching**: Employs a robust `localStorage` caching layer (`mp_crypto_chips`, etc.) with a 90-second Time-To-Live (TTL) to prevent severe API rate limiting on page reload.
- **Lazy Loading Integration**: Delays API fetch sequences until the DOM node actually intersects the viewport to optimize initial application load times.

## 3. Detailed Logic
- **Data Hydration (`loadCrypto`, `loadStocks`)**:
  - Checks `getCache()` first. If valid, renders immediately.
  - Opportunistically inspects the global `window.latestCryptoData` namespace to reuse fetches already completed by the primary Hero modules instead of executing redundant CoinGecko calls.
  - Employs `Promise.allSettled()` to iterate through `fetchSingleQuote(sym)` simultaneously for batches of up to 9 traditional stock/commodity symbols, ensuring one failed quote doesn't break the entire grid array.
- **Fail-Safes (Static Fallbacks)**:
  - If a fetch fails entirely or internet is disconnected, the system automatically injects hardcoded fallback arrays (e.g. `BTC: 96500`) allowing the UI to remain visually populated and structurally intact.
- **DOM Factory (`buildChip`)**:
  - Dynamically synthesizes `<a>` HTML elements injecting CSS classes computed by `changeClass()` (returning `mp-up`, `mp-down`, or `mp-neutral`).
  - Sets custom `--mp-delay` CSS variables mathematically offset by `(i % 9) * 0.35` seconds to create a cascading visually staggered entrance animation.

## 4. User Flow
1. User scrolls down on the Clarity Box / Dashboard.
2. The `IntersectionObserver` triggers, applying `.mp-visible` to fade in the grid.
3. System fetches/shows Cached or Live Crypto chips by default.
4. User clicks the 'Stocks' or 'Commodities' tab segment.
5. System displays a 9-block skeleton loading animation (`showSkeleton`) while fetching new Yahoo Finance endpoints in the background, then cascades the loaded chips into view.
6. User clicks a specific chip, redirecting them linearly to the dedicated `market-detail.html` screen passing the specified symbol via absolute payload limits.

## 5. Inputs / Outputs
- **Inputs**: `COINGECKO_MARKETS_URL`, global `fetchSingleQuote` resolver, DOM button clicks via `dataset.cat`.
- **Outputs**: Rendered HTML elements (`.mp-chip`), formatted string scalars scaling decimals correctly via `formatPrice()`, cache objects deployed to memory storage.

## 6. Edge Cases
- **Missing Global Functions**: Wrapped safely via `typeof fetchSingleQuote !== 'function'` checks terminating execution natively and deploying fallbacks if the primary global `script.js` failed to compile.
- **Rate Limited APIs**: Handles `fetch()` rejections natively through `catch (e)` blocks silently routing into offline backup arrays avoiding destructive console loops.

## 7. Dependencies
- **script.js / Global Dependencies**: Completely dependent on `fetchSingleQuote()` mapping for Yahoo endpoint translations and CORS routing.
- **CurrencyConverter**: Optionally leverages the global formatting system if present to map localized currencies properly.

## 8. UI / UX Behavior
- **Grid Layout**: Employs a strict 3x3 array (9 chips) responsive CSS grid prioritizing maximum data density.
- **Color Signatures**: Mutes standard numbers into gray values, only highlighting the delta `%` changes in standard Red/Green hues to reduce visual anxiety.
- **Accessibility**: Defines semantic `role="listitem"` and highly descriptive `aria-label` concatenations on the dynamic anchors for screen-readers.

## 9. Future Improvements
- Implement WebSocket streams for the Crypto module specifically to allow chips to lightly "flash" real-time adjustments instead of relying purely on 90s polling intervals.
- Expand localization, permitting specific region selections for the Stocks arrays (e.g. switching default US tech mappings for European Indices).

## 10. System Role
Acts as the immediate "at-a-glance" heartbeat of the application's home pages. It bridges the gap between educational content and the live financial reality outside the app.
