# Feature: Portfolio Manager

## 1. Purpose
Enables users to construct and monitor a multi-strategy paper portfolio. It enforces a deterministic allocator that assigns capital weights based on strategy lifecycle tiers and Composite Readiness Scores (CRS).

## 2. Core Functionality
- **Allocation Engine**: Automatically distributes capital among eligible strategies using a rules-based allocation algorithm.
- **Eligibility Filtering**: Restricts portfolio inclusion to strategies that meet minimum CRS or specific lifecycle tiers (e.g., VALIDATED, PRODUCTION).
- **Equity Aggregation**: Computes a unified portfolio equity curve and PnL by scaling individual paper trading session results.
- **State Management**: Persists portfolio configuration, slot assignments, and risk limits to `localStorage` (`pp_portfolio_v1`).

## 3. Detailed Logic
- **Slot Management**: Limits the portfolio to a maximum of 10 strategy "slots" (`MAX_SLOTS = 10`). Prevents duplicate strategy/version insertions.
- **Allocator Algorithm (`runAllocator`)**:
  - Distinguishes between `MANUAL` and `AUTO` weight modes. 
  - Caps manual allocations first based on `max_single_weight_pct`.
  - Distributes the remaining weight to `AUTO` slots based on an allocation score: `CRS + Tier Bonus` (Production = +20, Validated = +10).
  - Employs iterative redistribution (up to 10 passes) to reallocate excess weight when an `AUTO` slot breaches the maximum single weight cap.
- **Equity Computation (`getPortfolioEquity`)**:
  - Matches active portfolio slots with historical paper sessions from `pp_paper_sessions_v1`.
  - Scales individual session equity curves proportionately to the allocated capital.
  - Aggregates the scaled point-in-time equity values into a synchronized portfolio curve and calculates the maximum portfolio drawdown (MaxDD).

## 4. User Flow
1. User creates a paper portfolio, setting starting capital, max DD limits, and max single string exposure.
2. User adds verified strategies to the portfolio from the Strategy Library.
3. System validates eligibility (min CRS / Validated status).
4. System automatically re-weights the portfolio.
5. User monitors the unified equity curve, PnL, and individual contribution in the Portfolio Overview UI.

## 5. Inputs / Outputs
- **Inputs**: `localStorage` (`pp_portfolio_v1`, `pp_paper_sessions_v1`), strategy updates from `StrategyLifecycle`.
- **Outputs**: Calculated allocations (weights per slot), aggregated equity metrics (Total PnL, Portfolio DD, unified curve map), dynamic UI rendering for the Portfolio Modal.

## 6. Edge Cases
- **No Eligible Strategies**: Allocator successfully handles 0 eligible strategies by reducing all target weights to 0.
- **All Slots Capped**: If iterative redistribution caps all auto slots without reaching 100% total weight, the remaining excess conceptually sits as unallocated "cash".
- **Corrupted Strategy Data**: Validates and assigns fallback names (`Strategy #1`, etc.) upon loading corrupted portfolio states.

## 7. Dependencies
- **StrategyLifecycle**: Required to verify strategy versions, read CRS metrics, and resolve tiers.
- **PaperExecution / localStorage (`pp_paper_sessions_v1`)**: Required for extracting individual strategy equity curves to build aggregated metrics.
- **PortfolioAttribution & PortfolioRiskOverlay**: Downstream modules that rely on the Portfolio Manager's computed state.

## 8. UI / UX Behavior
- **Overview Tab**: Displays hero metrics (Total Equity, PnL, DD) with color-coded safety indicators.
- **Allocation Table**: Lists current strategies with explicit weights, states, and toggle controls.
- **Modal Overlay Structure**: Uses a full-screen, tabbed modal to act as an isolated portfolio workspace within the application.
- **Toast Notifications**: Reusable, transient popups provide non-blocking feedback for allocation updates and errors.

## 9. Future Improvements
- Implement temporal alignment for disparate strategy paper session timestamps during aggregation.
- Add support for multiple portfolios (currently limited to a single unified portfolio).
- Add support for fractional rebalancing (drift tracking over time).

## 10. System Role
Acts as the central integration point between individual strategy backtests/paper sessions and holistic wealth management, setting the stage for institutional-grade attribution and risk oversight.
