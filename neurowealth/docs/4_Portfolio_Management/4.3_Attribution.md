# Feature: Portfolio Attribution & Diagnostics

## 1. Purpose
Computes post-execution attribution metrics, strategic correlations, and diagnostic problem alerts for the portfolio. It identifies high-concentration risks, pinpointing precisely which strategies are driving returns and which are causing drags.

## 2. Core Functionality
- **Performance Attribution**: Calculates contribution return percentage, max drawdown, and rolling returns for every slotted strategy individually.
- **Diagnostic Synthesis**: Computes aggregated portfolio diagnostics, ranking the 'top contributor', 'worst drag', 'most underwater', and 'most stable' assets.
- **Correlation Engine**: Mathematically computes the Pearson correlation matrix between strategy returns to detect weak diversification.
- **Problem Detector**: Outputs plain-text diagnostic bullet points alerting users to critical structural issues in the portfolio configuration.

## 3. Detailed Logic
- **Slot Metrics Calculation**:
  - `contribReturnPct`: Computed by comparing weighted slot PnL against the portfolio's absolute starting capital.
  - Drawdown calculation: Manages localized peak-to-trough calculations strictly within the sub-segments of the slot's independent equity curve mapping.
  - Generates status tagging (`DRAGGING`, `HEALTHY`, `CRITICAL`) using mathematical heuristics based on drawdown depth > 15% and length.
- **Correlation Array (`computeCorrelationMatrix`)**:
  - Distils up to 10 strategies into standard normalized returns arrays (`(current/previous) - 1`).
  - Iterates against all strategy combinations executing standard Pearson Correlation computation: `[cov(x,y) / (sigma_x * sigma_y)]`.
- **Overlap Drawdowns (`computeDrawdownOverlap`)**:
  - Matches the boolean condition of a 1% drawdown window across multiple strategies strictly per time index to identify simultaneous compounding risk windows.
- **Diagnostic Scorer (`generateProblemBullets`)**:
  - Evaluates metrics like total DD concentration fraction. Example: If standard `risk_concentration` mathematically accounts for >60% of absolute portfolio DD, an alert is structurally emitted.

## 4. User Flow
1. User navigates to the 'Overview' tab in the Portfolio Manager.
2. System autonomously calls `PortfolioAttribution.computeAttribution()` passing the currently rendered state object.
3. User reviews the "Problem Detector", identifying immediately if correlation matrices or overlapping drawdowns warn against deployment. 
4. User clicks 'Export JSON' to download a timestamped, highly granular metadata readout of the portfolio session.

## 5. Inputs / Outputs
- **Inputs**: Aggregated portfolio and underlying session equity structure (`portfolio`, `eq`), via `PortfolioManager`.
- **Outputs**: Detailed JavaScript `metrics` object mapping attribution statistics per individual slot, problem bullets as a string array, and mathematical correlation matrices (`1.0` / `-1.0`).

## 6. Edge Cases
- **Insufficient Data**: Handled elegantly via bounds checks. Reverts correlation array and overlap functions natively if the portfolio holds < 2 active slotted strategies.
- **Zero Divisors**: Safeguards math division internally in Pearson algorithms if standard deviation variance equals absolute zero.
- **Corrupted Curves**: If `equityCurve` mappings are malformed, calculations fall back gracefully to 0.

## 7. Dependencies
- **PortfolioManager**: Only dependent structurally on the execution result object returned natively via `getPortfolioEquity()`.

## 8. UI / UX Behavior
- **JSON Exporter**: Dynamically creates and clicks an ephemeral `<a>` HTML tag holding `data:text/json;charset=utf-8` representing the full localized snapshot.
- Native integration directly onto the primary UI container in `pf-attribution-section` inside the modal DOM elements.

## 9. Future Improvements

## 10. System Role
Functions cleanly as a pure stateless math service module. It exists externally to the data-storage loop and solely ingests data to output advanced numerical intelligence.
