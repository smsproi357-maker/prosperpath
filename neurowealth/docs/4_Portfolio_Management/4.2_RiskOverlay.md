# Feature: Portfolio Risk Overlay

## 1. Purpose
Acts as a safety autopilot for paper portfolios. It autonomously monitors aggregate portfolio drawdowns, exposure limits, and underlying strategy health, stepping in to de-risk or halt trading if strict safety thresholds are breached.

## 2. Core Functionality
- **Drawdown-Based Intervention**: Implements a three-tier response mechanism (Warn, De-risk, Pause) based on portfolio-wide current drawdown.
- **Strategy Health Enforcement**: Periodically sweeps strategies to disable them if their Composite Readiness Score (CRS) drops below a defined floor or if stability drift is detected.
- **Exposure Limiting**: Instantly overrides strategy weights if they exceed the absolute maximum single-asset weight cap.
- **Audit Logging**: Maintains a chronological audit trail (last 200 events) of all automated risk interventions and parameter breaches.

## 3. Detailed Logic
- **State Machine Iteration (`evaluateOverlay`)**:
  - Evaluates current `portfolioDDPct` against predefined limits (`warn_dd_pct`, `soft_dd_pct`, `hard_dd_pct`).
  - **WARN**: Triggers visual UI banners only.
  - **DERISK**: Mathematically forces down the exposure of all enabled slots to a maximum combined portfolio weight of 50%. Calculates a `scaleFactor` applied dynamically as `overlay_weight_scale`.
  - **PAUSED**: Hard halts all enabled strategies (sets `overlay_weight_scale = 0`). Calculates an automatic cooldown timeout (`pause_minutes`), unless configured for mandatory manual resumption.
- **Slot Disable Checks**:
  - Checks if strategy `crs_snapshot` < `min_crs_to_stay_enabled`.
  - Evaluates recursive auto-pause events (using the audit log array) to permanently disable erratic strategies (e.g. `excessive_auto_pauses`).
  - Verifies alignment against the `StrategyHealth` class to fire the "drift kill switch" if a strategy is flagged as UNSTABLE.

## 4. User Flow
1. User enables the Risk Overlay in the Portfolio UI.
2. User sets max risk thresholds for Warn, De-risk, and Hard Pause states.
3. System silently monitors the portfolio configuration on every state change.
4. If a threshold is hit (e.g. 18% DD), the system transitions to `DERISK`, halving position sizes and injecting warning banners.
5. If DD recovers, the system seamlessly transitions back to `NORMAL`. If DD worsens, it transitions to `PAUSED`.

## 5. Inputs / Outputs
- **Inputs**: Real-time aggregated portfolio equity curve (`PortfolioManager.getPortfolioEquity()`), `localStorage` (`pp_portfolio_risk_policy_v1`), strategy CRS variables.
- **Outputs**: Derived `overlayState` (status mappings of each slot), adjusted multiplier weights (`overlay_weight_scale`), unified audit log (`pp_portfolio_risk_audit_v1`).

## 6. Edge Cases
- **Cooldown Expiry Race Conditions**: Manages cool-down durations explicitly through absolute timestamp logic (`isoNow()`) to prevent immediate re-trigger loops upon portfolio refresh.
- **Missing `StrategyHealth` Module**: Features a `try/catch` encapsulation around the drift kill switch check to ensure the overlay functions optimally even when strategy health analytics are partially loaded.

## 7. Dependencies
- **PortfolioManager**: Heavily dependent on the `getPortfolioEquity()` readout and current `.holdings` state.
- **PaperExecution**: Emits structured risk overlay audit events natively to the paper execution UI console string via `PaperExecution.captureEvent()`.
- **StrategyHealth**: Optional check dependency to enact drift kill switches.

## 8. UI / UX Behavior
- **Overlay Injection**: Employs a `MutationObserver` to dynamically inject the Risk Overlay UI directly into the Portfolio Manager Modal interface when it becomes visible.
- **Status Badges**: Colors and updates specific visual badges for every single strategy slot indicating if it is `ENABLED`, `DISABLED (policy)`, or `PAUSED`.
- **Audit Log Panel**: Renders a compact terminal-like list displaying the most recent 20 automated risk actions.

## 9. Future Improvements
- Implement volatility-based (VIX) de-risking thresholds alongside simple absolute drawdown numbers.
- Allow configuring different scaling parameters during 'DERISK' (e.g. scaling to 25% vs locked 50%).

## 10. System Role
The Risk Overlay represents the highest tier of the portfolio execution stack, serving as the ultimate institutional circuit breaker over aggregated simulation logic.
